Indicadores do Dashboard — o que cada KPI significa
📦 Exposição Total do Portfólio
🗄️ SQL View: vw_kpi_exposicao
SUM(exposicao_total) — último mês disponível
Valor total em risco no portfólio corporativo. No recorte base 12M, o portfólio soma R$ 9,49 bi.
✓ leitura estrutural do tamanho da carteira
🛡️ % Exposição Descoberta
🗄️ SQL View: vw_kpi_exposicao
exposicao_descoberta ÷ exposicao_total × 100
Mostra a parcela da carteira sem cobertura forte. No case, 25,0% ficou dentro da meta definida (< 30%).
✓ Meta: < 30%⚠ 30–35%✗ > 40%
⭐ Score Médio
🗄️ SQL View: vw_kpi_ratings
AVG(score_interno)
Indicador médio de qualidade de crédito do portfólio. Permite acompanhar deterioração ou recuperação da carteira ao longo do tempo.
✓ ≥ 800⚠ 750–799✗ < 650
📉 PD Média (ponderada por exposição)
🗄️ SQL View: vw_kpi_ratings — campo pd_ponderado_pct
SUM(pd_12m × exposicao_total) / SUM(exposicao_total) × 100
Probabilidade de default ponderada pelo tamanho de cada cliente no portfólio. Evita que clientes pequenos com PD elevada distorçam a leitura da carteira. A versão simples (AVG) está disponível como pd_medio_simples_pct nas views SQL para comparação.
✓ < 1%⚠ 1–2%✗ > 3%
⚠️ Operações Vencidas
🗄️ SQL View: vw_operacoes_vencidas
COUNT(*) WHERE status_operacao = 'Vencida'
Mede risco materializado. O objetivo é evidenciar atrasos, priorização de cobrança e necessidade de renegociação.
✓ 0⚠ 1–5✗ > 10
💰 Valor Vencido
🗄️ SQL View: vw_operacoes_vencidas
SUM(valor_utilizado) WHERE status_operacao = 'Vencida'
Complementa a contagem de operações vencidas com o impacto financeiro efetivo do atraso no recorte analisado.
✓ 0% da carteira⚠ até 1%✗ acima de 1%
💳 Utilização Média de Limites
🗄️ SQL View: vw_kpi_limites
AVG(valor_utilizado ÷ valor_limite) — base tratada
Foi ajustada para refletir a lógica da base tratada. Nesta versão: 65,0%, preservando coerência com os demais KPIs do projeto.
✓ 60–75%⚠ 75–85%✗ > 85%
🏦 Concentração Top 5
🗄️ SQL View: vw_kpi_exposicao
SUM(exposição dos 5 maiores clientes) ÷ exposição total × 100
Ajuda a avaliar o nível de pulverização da carteira e a dependência do portfólio em poucos nomes relevantes.
✓ até 35%⚠ 35–45%✗ acima de 45%
🏭 Concentração Setorial
🗄️ SQL View: vw_exposicao_por_setor
exp_setor ÷ exp_total × 100
Identifica excesso de concentração por setor frente ao limite regulatório de 15%.
✓ < 10%⚠ 10–15%✗ > 15%
🏛️ Provisão Total Estimada (EL)
🗄️ SQL Views: vw_kpi_exposicao · vw_stage_exposicao_recente
SUM(exposicao_descoberta × pd_ponderada_por_exposicao) — LGD implícito = 100%
Expected Loss estimada para a parcela descoberta da carteira, usando PD ponderada por exposição. O LGD implícito de 100% é conservador — colaterais reduzem a perda efetiva em cenários reais. Para fins regulatórios, a provisão por rating (CMN 2.682) está calculada em vw_stage_exposicao_recente.
✓ referência regulatória: % PCLD por rating (CMN 2.682)⚠ EL estimada com LGD=100% — premissa conservadora
🚨 Clientes em Atenção / Alto Risco
🗄️ SQL View: vw_clientes_em_atencao
Atenção operacional: util > 75% OU desc > 25% · Alto risco do case: util > 80% E desc > 30%
O dashboard foi ajustado para explicitar as duas camadas: a operacional, útil no dia a dia, e a regra estrita do case para aderência ao enunciado.
⚠ monitoramento operacional✗ regra crítica do case
🗄️ Fonte de Dados e Pipeline Analítico

Os dados partiram de um arquivo Excel (.xlsx), foram tratados em ambiente SQL Server para validação e depois convertidos para Amazon Athena como arquitetura-alvo. O HTML segue como wireframe navegável do QuickSight.

1. Excel
72 clientes, 222 operações, 864 ratings, 92 limites e 432 exposições.
2. SQL Server
Prototipagem, deduplicação, PK/FK e validação das regras do case.
3. Athena
Conversão das views para Presto/Trino, preservando a camada semântica.
4. QuickSight
Arquitetura-alvo. Este HTML representa o mockup interativo da futura publicação.
Leitura recomendada: use esta versão sem tags técnicas para apresentação executiva. Se precisar da versão técnica, basta alterar a constante SHOW_TECH_TAGS no script.
